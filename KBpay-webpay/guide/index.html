기본규칙

1. DTD & Language 선언

  • <!DOCTYPE html>
    <html lang="ko">

2. 인코딩

  • <meta charset="UTF-8">

3. 브라우저 호환성

  • IE(9~11), Chrome, Firefox
    ※개발관련 스크립트 호환성은 퍼블리싱 가이드에서 제외됩니다.

4. 최적화 기준 해상도

  • 최적해상도 : iPhone 6, 6s+, 7, 8 (개발 기준 해상도 375 x auto)
    320 ~ 414까지 고려함

5. 폴더구성

  • css style.css Style Source Import
    layout.css 레이아웃
    component.css 공통 구성요소 탭, 아코디온, 게시판 등
    images
    js jquery-2.2.4.min.js
    ui.js UI공통기능 정의
    util slick.css swiper 기능 swiper
    slick.min.js swiper 기능
    placeholder.js placeholder 기능

4. 네이밍 규칙

    • 1. css class명은 "-" (하이푼) 으로 구분지으며 직관적이고 길어지지 않도록 작명합니다.
      (네이밍룰 엑셀 참고)
    • 2. UI 기능구현을 위한 css (selector) 사용은 카멜표기법을 사용합니다.
      ex) closeLayer, openLayer
    • 3. css 작성 혹은 기능구현을 위한 셀렉터를 생성할때 ID 사용을 지양합니다.
      (ID는 Front-end 혹은 Back-end에서 사용할 수 있도록 합니다.)
      단 팝업아이디는 사용 가능
    • 4. 비슷한 유형의 타입을 추가 생성할 경우 최초 생성자에는 넘버를 붙이지 않고 2번째부터 사용합니다.
      ex) txt-type, txt-type02, txt-tytpe03, ...

3. 작성규칙

    • 1. HTML 마크업을 할때 DTD에 준수하는 표준 문법을 사용하며 문법오류, 중첩 오류를 체크합니다.
    • 2. Table을 사용할때 레이아웃과 표(Table) 구분을 명확하게 합니다.
      (표를 의미하는 경우 caption or summary제공, thead, tbody, scope 속성 모두 제시할 것 / 레이아웃으로 제공할 경우에는 반대로 cation, summary, scope 속성 제거)
